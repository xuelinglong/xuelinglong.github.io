深拷贝与浅拷贝

指的是对于Object、Array这种引用类型，当从一个变量复制引用类型的值时，复制的其实是一个指针，两个变量指向同一个堆对象，改变其中一个变量，另一个也会受到影响。

浅拷贝：拷贝引用。

深拷贝：拷贝实例(新的实例与之前的实例互不影响)。

浅拷贝包括：Array.prototype.slice()，Array.prototype.concat()，

深拷贝常用方法：JSON.parse(JSON.stringify(obj))、fon…in加递归

JOSN对象中的stringify可以把一个js对象序列化为一个JSON字符串，parse可以把JSON字符串反序列化为一个js对象，通过这两个方法，也可以实现对象的深复制。

Promise的finally、all、race等方法

finally是不管promise对象最后状态如何，都会执行的操作;finally方法的回调函数不接收任何参数，所以finally里面的操作是与promise状态无关的。

all用于将多个promise实例包装成一个新的promise实例；接受一个数组（内部是一个个的promise实例，若不是则会调用resolve方法转换为promise实例再处理）为参数；只要有一个promise被rejected那么p就会变成rejected状态（第一个被rejected的promise实例的返回值会传递给p的回调函数），必须所有的promise都fulfilled那么p才会fulfilled（所有promise实例的返回值组成一个数组传递给p的回调函数）。

const p = Promise.all([p1, p2, p3])

race也用于将多个promise实例包装成一个新的promise实例；接受一个数组（内部是一个个的promise实例，若不是则会调用resolve方法转换为promise实例再处理）为参数；只要有一个promise改变状态那么p就会改变状态，且率先改变状态的promise实例的返回值会传递给p；

const p = Promise.race([p1, p2, p3]);

all与race的相同点：

  1. 都用于将多个promise实例包装成一个新的promise实例；
  2. 如果参数数组中有非promise实例的存在，都会先调用Promise.resolve方法转换为promise；

all与race的区别：

  1. 状态的改变方式不同
  2. 传递给回调函数的值不同

跨域的方法

同源需满足：域名相同 + 协议相同 + 端口相同。否则跨域请求的返回结果会被浏览器拦截掉。

缓存方案

强缓存：Cache-Control：max-age定义了一个最大使用期。

协商缓存：需要和服务器进行协商是否使用本地缓存。

浏览器缓存

1.localstorage：可无视用户主动刷新页面，并且可储存较大体积的资源(2M以上)，setItem存getItem取。

2.sessionStorage：关闭窗口后失效，setItem存getItem取。

场景：页面滚动，将滚动位置存到session中 -> 再次进到页面中，到session中取出上次保存的浏览位置 -> 滚动到对应位置。

http协议、contentType

兼容性问题

浏览器兼容性问题分为：样式兼容性(css)、交互兼容性(javascript)、浏览器hack

1.样式兼容性(css)

可通过Normalize.css 抹平差异、或者定制自己的 reset.css、或者通过通配符选择器，全局重置样式* { margin: 0; padding: 0; }

添加浏览器前缀：IE-ms、Firefox-moz、Opera-o、Chrome/Safair-webkit。

IE9以下透明属性不兼容，可通过filter: alpha(opacity = 50);

2.交互兼容性

非chrome浏览器获取scrollTop 通过 document.documentElement.scrollTop兼容：var scrollTop = document.documentElement.scrollTop||document.body.scrollTop;

3.浏览器hack

判断是否是Safari浏览器var isSafari = /a/.__proto__=='//';

判断是否是Chrome浏览器var isChrome = Boolean(window.chrome);

原生ajax请求

一个完整的ajax请求包括五个步骤:
　　1.创建XMLHTTPRequest对象

　　2.使用open方法创建http请求,并设置请求地址

　　3.设置发送的数据，开始和服务器端交互

　　4.注册事件

　　5.获取响应并更新界面

防抖是什么

场景描述：用户拖拽改变窗口大小过程中窗口的大小一直在改变，如果在resize事件中绑定函数，这个函数将会一直触发，无意义且会浪费资源，此时可用函数防抖来优化。

防抖就是在短时间内多次触发同一个函数，只执行最后一次，或者只在开始时执行。

特性：触发了一个事件后：如果有一个定时任务待执行，就清除定时器，重新计时；如果没有任务待执行，就定时执行这个事件。

性能优化的方案

网络方面：减少http的请求次数，比如将多个文件打包成一个；减少资源文件的体积；缓存

渲染和操作DOM方面：将css文件放在头部，js文件放在底部；减少对DOM的直接操作，减少定时器的使用；

数据方面：图片的预加载；首屏加载进度条的显示；使用JSON进行数据交互，对部分常用数据进行缓存，数据埋点和统计。
